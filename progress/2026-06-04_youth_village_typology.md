# 2026-06-04 — 청년마을 카탈로그 + 새 유형 시스템 (자세 메인 + 환경 부제)

## 결정 요약

### 1. 유형 시스템 구조
**자세(Stance)가 메인**, **환경(Env)이 부제**.

이유:
- 환경 선호는 사용자가 이미 알고 있어서 진단 임팩트 적음
- 자세는 "오, 나 이런 사람이었구나" 발견의 재미가 있음
- 추천 점수에는 환경 가중치가 더 큼 (산 좋아하면 산 청년마을이 강력 추천) — 표시와 별개

```
👤 결과 카드 예시
   사랑방형 · 같이 + 쉼   ← 메인 (자세)
   🌊 바다·해안           ← 부제 (환경)
```

### 2. 자세 4유형 (가제)

| 코드 | 가제 | 톤 |
|---|---|---|
| `alone_rest` | 산책자형 / 피톤치드형 / 비우는사람형 | 혼자 조용히 비우기 |
| `alone_make` | 작업실형 / 손쟁이형 / 짓는사람형 | 혼자 손으로 만들기 |
| `together_rest` | 사랑방형 / 마실형 / 동네벗형 | 사람들과 어울리며 천천히 |
| `together_make` | 마을기획자형 / 동네짓기형 | 동네에 무언가 보태기 |

이름은 **가제** — 의미 ID 기반이라 변경 시 meta 한 줄 수정으로 끝.

| `sea` | 🌊 바다·해안 | 강화·영덕 |
| `field` | 🌾 들·평야 | 세종·의성 |
| `village` | 🏘 도시·골목 | 홍성·대전 |

### 4. 추천 점수 산출

```
환경 일치  → +60점
자세 일치  → +30점
자세 보조 일치 (stanceAlt) → +15점
같은 sociality 또는 같은 mode 부분 일치 → +5점씩
```

라벨링:
- **strong** (≥80점) → "강력 추천 ⭐"
- **good** (50~79점) → "이 분위기도 어울려요"
- **alt** (25~49점) → "다른 풍경도 한번"
- **weak** (<25점) → 노출 X

## 결정 메모

### 청년마을 카탈로그 확정 과정
- 처음엔 산/바다 위주로 잡았다가 들녘 청년마을이 적다는 한계 인지
- 들 청년마을이 진짜 적음 — 세종 농땡이월드(연서), 의성 나만의성으로 채움
- 영월 밭멍은 산악이지만 "밭" 컨셉이라 처음엔 들로 분류 → 다시 산으로 (밭멍의 산악 특성 반영)
- 무주 산타지, 의성 나만의성의 stance는 이름의 첫인상과 반대 (사용자가 정확한 정보로 정정)

### 자세 메인 + 환경 부제로 가는 이유
- "산사람형 / 바다사람형" 같은 이름은 사용자가 자기 답을 알고 있어 진단 임팩트가 죽음
- "마을기획자형"처럼 자세 기반 이름이 캐릭터로 더 강력
- 추천 가중치는 여전히 환경이 크지만 (산 좋아하면 산 청년마을 강추), 사용자 인지에는 영향 없음

### "집"이 아니라 "나의 공간" 톤 (이전 결정 연장)
- 청년마을(레지던스)과 일차 진행 시각화(나의 공간 SVG)는 의미 분리
- 청년마을 = 실제 머무는 곳, 나의 공간 = 일차 진행의 상징
