# 2026-06-06 — 홈 화면 정체성 개편 ("나의 마음 자리")

## 멘토 피드백

> 지금 홈 화면은 홈 화면으로서 기능하는것처럼 보이진 않는다. 뭔가 홈에서 미션도 하고 하는 것이 홈화면의 정체성이 잘드러나지 않는다.

### 문제 진단
- 기존 홈은 사실상 **"떠나기 버튼" 한 개의 런처**. 페이지 정체성 없음.
- 사용자가 이 앱에서 무엇을 누적하고 무엇을 머무는지가 홈에서 안 보임.
- "잠시, 다른 지역의 바람을 짓고 와볼래요?" 코피가 떠나는 액션만 강조.

### 청풍 비전과의 어긋남
청풍 비전은 "관계 쌓기 + 미래 그리기" 인데, 기존 홈은 그저 출발점. **사용자의 "쌓임"이 보여야 함.**

### 섹션별 의도

| 섹션 | 의도 | 데이터 |
|---|---|---|
| **헤더** | "여기는 너의 자리야" 환영 | `nickname`, `homeRegion`, 다녀온 지역 수 |
| **내 마음의 자리** | 누적이 "눈에 보이게". 빈 자리부터 자리 잡기까지 시각화 | `HouseStage` 4단계, 라벨 `SPACE_STAGE_NAMES` |
| **오늘의 편지** | 매번 들어올 때 잠깐 머무를 거리 | `storiesByResidenceId` (최근 다녀온 곳 우선, 없으면 첫 추천) |
| **다녀온 흔적** | 가벼운 자랑/회상 | `regionProgress.visitCount > 0` 필터링한 residences |
| **새 지역 떠나기** | 액션이지만 흐름의 일부 | 기존 `onDepart` |

### 마음 자리 단계 매핑

```ts
function innerStageForVisited(count: number): 0 | 1 | 2 | 3 {
  if (count <= 0) return 0;  // 빈 자리
  if (count === 1) return 1;  // 자리 잡기
  if (count <= 3) return 2;  // 깊어짐
  return 3;                   // 내 자리
}
```

`HouseStage` 컴포넌트(4단계 SVG)와 `SPACE_STAGE_NAMES` 라벨은 잠시섬 일차 시각화용으로
이미 만들어둔 것 그대로 재사용 → 톤 일관.

## 유저 시나리오

### 첫 진입 (방문 0회)
- 헤더: "{닉네임}님의 마음 자리 / 서울에서 시작해, 지금까지 0곳 다녀왔어요"
- 마음 자리: "빈 자리" (stage 0, SVG 빈 터). 카피: "아직 빈 자리예요. 떠나면서 한 자리씩 채워질 거예요."
- 오늘의 편지: 추천 첫 지역(강화도)의 Story 미리보기 — 빈 홈에서도 "맛보기"
- 다녀온 흔적: 섹션 자체 숨김
- 떠나기 카드: 그대로 노출

### 1~3곳 다녀온 후
- 헤더 카운트가 1·2·3으로 갱신
- 마음 자리: stage 1(자리 잡기) → stage 2(깊어짐) → SVG가 한 단계씩 자라남
- 오늘의 편지: 다녀온 곳 중 첫 번째의 Story
- 다녀온 흔적 칩이 누적

### 4곳+
- 마음 자리: stage 3(내 자리). 한옥 처마 + 등불 + 굴뚝 연기 까지 완성된 SVG
- "여기는 진짜 내 자리"의 시각적 도달감
